## 3.5-1 Explicit Pipelines & Abuse Prevention

### Status
Compliant (Strong)

### Security Basis
- Abuse resistance
- Supply-chain integrity
- Insider threat mitigation

### Design Guarantees
- All processing expressed as explicit DAGs
- No hidden or implicit processing nodes
- Full pipeline visibility before execution
- Pipelines are logged and auditable

### Security Outcome
Prevents silent transformations, hidden malicious logic,
and supply-chain abuse through undocumented processing.

### Action Required
None
